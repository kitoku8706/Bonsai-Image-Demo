# 🌳 Bonsai Image 4B: 로컬 기기용 1비트/터너리 이미지 생성 모델

> **출처:** [GeekNews (news.hada.io)](https://news.hada.io/topic?id=30073) | 원문: [prismml.com](https://prismml.com)

노트북과 휴대폰 같은 로컬 하드웨어에서 고품질 확산(Diffusion) 추론을 실행하도록 설계된 **소형 이미지 생성 모델군**입니다.

### ✨ 주요 요약
- **아키텍처:** FLUX.2 Klein 4B 기반 (아키텍처 유지)
- **경량화:** 확산 트랜스포머 가중치를 **1-bit** 또는 **ternary** 표현으로 변환
- **메모리 절감:** 원본 7.75GB ➡️ 1-bit **0.93GB** / ternary **1.21GB**
- **속도:** 
  - iPhone 17 Pro Max (512×512): **9.4초**
  - Mac M4 Pro (512×512): **약 6초** (MFLUX 대비 최대 5.6배 빠름)
- **성능:** ternary 모델은 원본 FLUX.2 Klein 4B 대비 **95%** 성능 유지
- **오픈소스:** Apache 2.0 라이선스로 가중치 및 코드 공개 예정

---

## 🚀 로컬 이미지 생성을 위한 Bonsai Image 4B

Bonsai Image 4B는 기기 내부(온디바이스)에서 고품질 이미지 생성을 가능하게 하는 것을 목표로 합니다.
- **1-bit 모델:** 이진 `{-1, +1}` 가중치 + FP16 그룹 스케일링 팩터 (가중치당 1.125 유효 비트)
- **Ternary 모델:** `{-1, 0, +1}` 가중치 + FP16 그룹 스케일링 팩터 (가중치당 1.71 유효 비트)
  - 1-bit보다 크지만 '0' 상태가 추가되어 시각적 품질과 프롬프트 충실도가 더 뛰어납니다.
- **iPhone 최초 구동:** 해당 파라미터급 이미지 모델 중 iPhone에서 직접 실행되는 첫 모델입니다.

## 💾 로컬 실행을 위한 메모리 절감

로컬 이미지 생성의 가장 큰 장벽은 '기기 메모리'입니다. 가장 많은 메모리를 차지하는 '확산 트랜스포머'의 크기를 획기적으로 줄였습니다.
- **트랜스포머 크기:** FLUX.2 Klein 4B (7.75GB) ➡️ 1-bit (0.93GB) / Ternary (1.21GB)
- 이진(1-bit) 레이어는 약 14배, Ternary 레이어는 약 10배 크기를 줄였으며, 정밀도에 민감한 5%의 레이어만 FP16을 유지했습니다.

### 배포 페이로드 & 런타임 메모리
- **배포 크기 (Apple Silicon):** 1-bit (3.42GB), Ternary (3.88GB) *(원본: 15.97GB)*
- **활성 메모리 (512x512 기준):** 1-bit (1.5GB), Ternary (1.96GB) *(원본: 11.74GB)*

## ⚡ 지원 하드웨어 및 실행 성능

- **지원 환경:** Apple Silicon (iPhone, iPad, Mac - MLX 사용), CUDA GPU (Gemlite 커널 사용)
- iPhone 17 Pro Max와 같이 기존 모델은 메모리 부족으로 실행할 수 없었던 기기에서도 원활하게 온디바이스 실행이 가능합니다.

## 📊 벤치마크 성능 (원본 모델과의 비교)

- **GenEval, HPSv3, DPG-Bench** 3가지 벤치마크로 평가되었습니다.
- **Ternary 변형:** 원본 대비 **95% 성능** 유지 (크기 6.4배 감소)
- **1-bit 변형:** 원본 대비 **88% 성능** 유지 (크기 8.3배 감소, 1GB 미만 달성)
- SDXL, BK-SDM-Small, Stable Diffusion 1.5 등 비슷한 풋프린트를 가진 모델이나 더 큰 모델들보다 압도적으로 높은 성능을 보입니다.

## 💡 로컬 추론의 제품적 의미

클라우드 기반 생성은 서버 유지 비용이 들고, 매번 요청할 때마다 지연 시간이 발생합니다. 
반면, 로컬 추론은:
1. **비용 절감:** 실행 비용(서버비)이 들지 않음
2. **반복 속도 향상:** 네트워크 지연 없이 빠른 반복 생성 가능
3. **프라이버시:** 프롬프트와 결과물이 외부로 유출되지 않음

## 🔗 공개 방식 및 리소스

- iOS 앱인 **Bonsai Studio**도 함께 출시되어 iPhone에서 직접 체험해 볼 수 있습니다.
